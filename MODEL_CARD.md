# Model Card — RL Test Flow Optimization

## Overview

This model is a reinforcement-learning policy that selects an optimal sequence of semiconductor tests to maximize defect detection coverage while respecting wall-clock time and cost budgets. Best performing algorithm: **DQN** trained with Stable-Baselines3, evaluated on Kaggle T4 CPU across a 3-stage training funnel.

| Field | Value |
|---|---|
| Model type | Off-policy RL (DQN, MLP policy) |
| Framework | Stable-Baselines3 + sb3-contrib + Gymnasium |
| Input | 1003-dim vector: `tests_run_mask` (200) + `running_coverage` (200) + budget/time/chip features (603) |
| Output | Discrete action — test index (0–199) or STOP (200) |
| Training compute | ~6 h on Kaggle T4 CPU (3-stage funnel: 200K + 500K×3 seeds + 1M steps) |
| Inference latency | <20 ms per action on CPU |
| Trained on | Kaggle (T4 CPU, 19.5 GB RAM) |

## Intended Use

**Primary use case.** Optimize the test flow for a single chip at a time during post-silicon validation. Given a chip's feature vector and the remaining budget, recommend the next test that maximizes expected defect coverage per unit cost.

**Intended users.** Test engineering teams who need to cut characterization time per wafer without dropping coverage SLAs.

**Out-of-scope uses.** Wafer-level multi-chip optimization, pre-silicon verification, any workflow where test failures are correlated across chips.

## Training Data

- **Source.** Synthetic dataset generated by `generate_test_data.py` — 1,000,000 chips × 1,000 candidate tests with a 70% defect rate and 14 injected defect categories (electrical, timing, thermal, functional, analog).
- **Splits.** 800K train / 100K val / 100K test, stratified on defect category.
- **Environment reward shaping.**
  - `+1.0` × coverage gained by the selected test
  - `−0.1` × normalized test cost
  - `−0.05` × normalized test time
  - Episode terminates on budget, time, or test exhaustion.
- **Action masking.** Tests already run, tests that exceed remaining budget/time, and tests with zero marginal coverage are masked.

## Evaluation — Real Kaggle Execution Results

All results from actual Kaggle T4 CPU runs (April 2026). Environment: 200 tests, cost budget=50, time budget=120.

### Baselines (500 episodes each)

| Policy | Mean Reward | Accuracy | Mean Cost |
|--------|-------------|----------|-----------|
| Random | +3.35 | 86.8% | 48.52 |
| Greedy Coverage | +3.84 | 88.0% | 49.85 |
| Cost-Efficient | **+5.49** | 87.8% | 21.63 |

### Stage-1: 200K steps (4 algorithms)

| Algorithm | Mean Reward | Accuracy | Mean Tests | Train Time |
|-----------|-------------|----------|------------|------------|
| A2C | -413.41 | 56% | 1.0 | 9.1 min |
| **DQN** | **-28.42** | **89%** | **7.5** | **7.4 min** |
| PPO | -404.11 | 81% | 2.0 | 7.1 min |
| MaskablePPO | -411.84 | 60% | 1.0 | 8.9 min |

> Note: Stage-1 reward values reflect pre-evaluation-fix metrics. A2C/PPO/MaskablePPO suffered from deterministic duplicate-action loops (-1×410 per episode). DQN's off-policy Q-function naturally avoided this.

### Stage-2: 500K steps × 3 seeds (top-2 algorithms)

| Algorithm | Mean Reward | Std± | Accuracy | Mean Cost |
|-----------|-------------|------|----------|-----------|
| **DQN** | **+4.78** | 1.08 | **91.7%** | 38.90 |
| PPO | -3.86 | 1.30 | 59.0% | 3.98 |

DQN exceeded the best baseline (+5.49) in seed 777 (+5.70 reward).

| Metric | Value |

## Training Procedure

Three-stage training funnel:

1. **Stage 1 — Broad search** (`train_rl_kaggle_nb1.ipynb`): 4 algorithms × 200K timesteps. DQN selected as winner (highest accuracy 89%, only algo to beat random baseline).
2. **Stage 2 — Seed stability** (`train_rl_kaggle_nb2.ipynb`): Top-2 (DQN + PPO) × 500K timesteps × 3 seeds. DQN confirmed: mean +4.78, std=1.08.
3. **Stage 3 — HPO + Final** (`train_rl_kaggle_nb3.ipynb`): Optuna 30 trials × 100K steps → best params → 1M step final training.

Curriculum learning (Step 9): DQN trained progressively on n_tests=[10, 50, 200] × 200K each.

## Limitations

- Synthetic data — defect patterns may not match real fab correlations.
- Trained on a fixed 1,000-test catalog; adding new tests requires retraining (warm-start supported but not validated).
- Single-chip scope — does not exploit cross-wafer information.
- MaskablePPO's action mask assumes mask correctness; a buggy mask silently degrades coverage.

## Ethical Considerations

- No personal data — only simulated chip IDs and test results.
- Cost optimization may incentivize skipping low-yield tests that catch rare-but-critical defects. Production deployments should pin a floor-coverage constraint on safety-critical test categories.

## Reproducibility

```bash
# Generate data
python generate_test_data.py --chips 1000000 --tests 1000 --seed 42

# Train end-to-end
python -m src.cli train-funnel --algo auto --timesteps 1700000 --seed 42

# Evaluate
python -m src.cli evaluate --model models/best_model.zip --n-episodes 10000

# Serve
uvicorn src.api:app --host 0.0.0.0 --port 8000
```

All randomness controlled by `SEED=42` in `.env`. MLflow run logs include full hyperparameter set and environment config.
